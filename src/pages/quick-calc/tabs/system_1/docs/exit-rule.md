## 核心原则

> 反转交易的起点和终点都是明确的：从一个关键位到另一个关键位。
> 出场逻辑围绕这一点设计：错了就走，对了就拿到目标。

---

## 一、出场类型总览

| 优先级 | 出场类型     | 触发条件                   | 性质                   |
| ------ | ------------ | -------------------------- | ---------------------- |
| 1      | 止损出场     | 价格触及止损价             | 反转失败，保命         |
| 2      | BTC 反转出场 | BTC 方向变为不利           | 环境变了，防御         |
| 3      | 时间止损     | 持仓超过 10 天未达 1R      | 判断可能错了，释放资金 |
| 4      | 保本出场     | 利润曾达 1R 后回落至保本位 | 没亏，但没到目标       |
| 5      | 目标止盈     | 价格到达下一个反向关键位   | 反转成功，主动收割     |

---

## 二、止损出场（优先级 1）

入场时已设定止损价（见 entry-rules.md），价格触及即平仓。

| 方向 | 止损价            |
| ---- | ----------------- |
| 做多 | 关键支撑位 × 0.98 |
| 做空 | 关键阻力位 × 1.02 |

- **止损单必须在入场时同时挂好**，不能靠手动盯盘
- 止损只能向有利方向移动，永远不能往不利方向移动
- 触发后不犹豫、不补仓、不"再看看"

---

## 三、BTC 反转出场（优先级 2）

持仓期间，每天 UTC 收盘后检查 BTC 方向。

| 你的持仓 | BTC 变成                           | 操作               |
| -------- | ---------------------------------- | ------------------ |
| 做多     | 弱势（收盘价 < MA50 且 MA50 下行） | 次日平仓           |
| 做空     | 强势（收盘价 > MA50 且 MA50 上行） | 次日平仓           |
| 任何方向 | 震荡                               | 继续持有，不受影响 |

**为什么**：BTC 转向后，山寨币大概率跟随。继续持有等于逆势。

---

## 四、时间止损（优先级 3）

入场后 10 天内 浮动利润未达到 1R → 平仓出场。

1R = 入场价到止损价的距离

示例：
入场价 60,600，止损价 58,800
1R = 1,800 点（价格需到 62,400 才算达 1R）

10天后价格 61,500（利润 900 < 1R）
→ 平仓，释放资金

**为什么**：反转交易如果判断正确，价格通常在几天内就会开始朝目标方向运动。10 天还没动，说明判断可能不对或时机不对。

---

## 五、保本止损（优先级 4）

### 触发条件：浮动利润 ≥ 1R

做多：止损上移至 → 入场价（保本）
做空：止损下移至 → 入场价（保本）

之后止损就固定在保本位，不再追踪。两种结局：

- 价格继续到达目标 → 止盈出场
- 价格回落到入场价 → 保本出场（不亏不赚）

**为什么不做追踪止损**：反转交易的目标明确（下一个关键位），从 A 到 B 的过程中价格经常震荡，追踪止损容易在震荡中被扫出去 — 方向对了却没赚到。保本止损 + 固定目标更适合这种风格。

---

## 六、目标止盈（优先级 5）

### 入场时确定目标

目标价 = 下一个反向关键位

| 方向 | 止盈目标                                                 |
| ---- | -------------------------------------------------------- |
| 做多 | 上方最近的阻力位（摆动高点 / MA50/200 / 成交密集区上沿） |
| 做空 | 下方最近的支撑位（摆动低点 / MA50/200 / 成交密集区下沿） |

### 止盈条件

价格到达目标关键位 ±1% 范围内 → 平仓止盈。

不要等精确触及目标价 — 差 1% 就够了，贪最后一点利润往往导致利润回吐。

### 盈亏比检查（入场前必须计算）

做多：盈亏比 = (目标价 - 入场价) ÷ (入场价 - 止损价)
做空：盈亏比 = (入场价 - 目标价) ÷ (止损价 - 入场价)

要求：盈亏比 ≥ 2

盈亏比 < 2 → **不做这笔交易**。说明目标位太近或止损太远，风险收益不划算。

---

## 七、出场决策流程

每天 UTC 收盘后，对每个持仓执行：

1. 止损触发了吗？→ 是 → 已自动平仓

2. BTC 方向变了吗？→ 对我不利 → 次日平仓

3. 持仓超过 10 天了吗？
   → 是，且利润 < 1R → 平仓
   → 是，但利润 ≥ 1R → 继续持有（已在保本位）

4. 利润 ≥ 1R 了吗？→ 是 → 止损移至保本位（只做一次）

5. 价格到达目标止盈位 ±1% 了吗？→ 是 → 平仓止盈

6. 以上都不是 → 继续持有，明天再查

## 九、完整交易示例（入场到出场）

【入场】
BTC 状态：强势（收盘 > MA50，MA50 上行）→ 允许做多
品种：ETH
关键支撑位：2,480（成交密集区下沿）
确认K线：锤子线 ✓
入场价：2,520
止损：2,480 × 0.98 = 2,430
1R = 2,520 - 2,430 = 90 点

目标：上方阻力位 2,800
盈亏比 = (2,800 - 2,520) / 90 = 3.1 ≥ 2 ✓ → 入场

【持仓管理】
第1天：价格 2,550（+30点，< 1R）→ 持有
第4天：价格 2,620（+100点，≥ 1R = 90）→ 止损移至 2,520（保本）
第5天：价格回落到 2,560 → 保本位未触及，继续持有
第7天：价格 2,720 → 接近但未到目标 2,800 ±1%，继续持有
第9天：价格 2,790（在 2,800 ±1% 范围内）→ 平仓止盈

【结果】
出场方式：目标止盈
出场价：2,790
盈利：2,790 - 2,520 = 270 点
R倍数：270 / 90 = 3.0R
持仓天数：9天

### 如果反转失败的示例

【入场】同上

【持仓管理】
第1天：价格 2,500（-20点）→ 持有
第2天：价格 2,440 → 接近止损但未触及
第3天：价格 2,425 → 触及止损 2,430 → 自动平仓

【结果】
出场方式：止损
出场价：2,430
亏损：2,520 - 2,430 = 90 点 = 1R
持仓天数：3天
亏损金额：10U（总资金的2%）

_反转交易的出场逻辑：错了快走（止损），对了拿到目标（止盈），中间只做一次保护（保本止损）。简单明确。_
