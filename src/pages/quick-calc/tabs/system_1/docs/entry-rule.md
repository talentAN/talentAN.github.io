## 核心原则

> 每一条规则都必须能回答"是"或"否"，不存在"差不多""感觉像"。

## 一、BTC 方向过滤（每日第一步）

在看任何币种之前，先判断 BTC 当前状态，决定今天**允许什么方向**：

| BTC 状态 | 判定条件                                     | 允许方向 |
| -------- | -------------------------------------------- | -------- |
| 强势     | BTC 日线收盘价 > MA50，且 MA50 上行          | 只做多   |
| 弱势     | BTC 日线收盘价 < MA50，且 MA50 下行          | 只做空   |
| 震荡     | 其他情况（收盘价在 MA50 附近，或 MA50 走平） | 多空都可 |

MA50 方向判断：MA50 当前值 vs 10 天前的值，差异 > 3% 为上行或下行，≤ 3% 为走平。

每天 UTC 收盘后，10 秒完成判断：

BTC 收盘价 vs MA50 → 在上方还是下方？
MA50 vs 10天前 → 涨了/跌了多少？

→ 得出今天允许的方向 → 再去看其他币种的信号

本质：不和 BTC 对着干。

## 二、入场决策流程

步骤0：BTC 方向过滤 — 确定今天允许做多/做空/都可
步骤1：标记关键位（每周日）
步骤2：UTC日线收盘后检查 — 当日K线触及关键位了吗？→ 否 → 不看了
步骤3：触及了 → 这根K线是合格的确认K线吗？→ 否 → 继续等
步骤4：是 → 方向符合 BTC 过滤吗？→ 否 → 放弃
步骤5：当日24h成交量 ≥ 200万U？→ 否 → 放弃本次信号
步骤6：定止损 → 算仓位 → 过检查清单
步骤7：下单（入场 + 止损同时挂）

### 步骤2 判断方法（当日K线是否触及关键位）

UTC 日线收盘后，用当日K线的最高价（H）和最低价（L）判断：

- **支撑位 P**：当日最低价 L ≤ P × 1.02 → 价格从上方接近或触及了支撑位
- **阻力位 P**：当日最高价 H ≥ P × 0.98 → 价格从下方接近或触及了阻力位

用最高/最低价而不是最新价，因为你看盘时价格可能已经弹回去了，但盘中确实到过关键位。

## 二、关键位的精确定义

### 类型 A：水平支撑/阻力位

首先定义"转折点"：

摆动高点（Swing High）：
一根K线的最高价 > 它前面2根K线的最高价
且 > 它后面2根K线的最高价
→ 价格在这里掉头向下了

摆动低点（Swing Low）：
一根K线的最低价 < 它前面2根K线的最低价
且 < 它后面2根K线的最低价
→ 价格在这里掉头向上了

注意：摆动高/低点需要后续2根K线来确认。最新的1~2根K线无法判定，但没关系 — 画线用的是历史数据，入场用的是确认K线形态。

#### 阻力位的标记条件

在日线图上，过去 90 根K线 内，存在一个价格 P，满足：

- 至少 1 个已确认的摆动高点 的最高价落在 P 的 ±1.5% 范围内

有 2 个以上摆动高点在同一价位（间隔 ≥5 根K线）的阻力位更强，但 1 个即可标记。

#### 阻力位的入场条件

标记了阻力位 P 之后，当价格**再次**到达 P ±2% 范围内，且日线收出确认K线（射击之星/看跌吞没/黄昏之星）→ 入场做空。

这个流程等效于"2次被弹回"：
第1次：历史摆动高点（已确认的转折）
第2次：当前价格到达 + 确认K线形成（正在发生的转折）

          P ─────────────────────
         ╱╲                  ╱╲
        ╱  ╲                ╱  ╲ ← 出现确认K线，入场做空
       ╱    ╲              ╱
      ╱      ╲            ╱
              ╲          ╱
               ╲        ╱
                ╲──────╱
        ↑ 第1次被弹回       ↑ 第2次正在被弹回
        （已确认摆动高点）    （确认K线验证）

举例：

BTC 在过去3个月内：
2月5日 最高价 72,800 → 是摆动高点（前后2根K线的高点都低于它）✓
→ 72,800 附近标记为阻力位（P ≈ 73,000）

    3月15日 BTC 涨到 73,200（在 P 的 ±2% 范围内）
    日线收出射击之星 → 确认信号 ✓
    → 入场做空

#### 支撑位的标记条件

在日线图上，过去 90 根K线 内，存在一个价格 P，满足：

- 至少 1 个已确认的摆动低点 的最低价落在 P 的 ±1.5% 范围内

有 2 个以上摆动低点在同一价位（间隔 ≥5 根K线）的支撑位更强，但 1 个即可标记。

#### 支撑位的入场条件

标记了支撑位 P 之后，当价格**再次**到达 P ±2% 范围内，且日线收出确认K线（锤子线/看涨吞没/早晨之星）→ 入场做多。

举例：

BTC 在过去3个月内：
3月1日 最低价 59,200 → 是摆动低点 ✓
→ 59,200 附近标记为支撑位（P ≈ 59,000）

    3月20日 BTC 跌到 59,500（在 P 的 ±2% 范围内）
    日线收出锤子线 → 确认信号 ✓
    → 入场做多

**关键位强度分级**（参考，非必须）：

| 强度 | 条件                                   | 说明                |
| ---- | -------------------------------------- | ------------------- |
| 普通 | 1 个摆动高/低点                        | 最低标记条件        |
| 较强 | 2 个摆动高/低点在 ±1.5%，间隔 ≥5 根K线 | 历史上已被弹回 2 次 |
| 很强 | ≥3 个摆动高/低点，或与 B/C 类型重合    | 多重确认            |

**实操方法**：

1. 打开日线图，缩放到能看到最近 90 根K线
2. 找到明显的"尖角"（价格掉头的位置）— 这些就是摆动高/低点
3. 在每个尖角的价位画水平线
4. 多个尖角在同一价位 → 更强的关键位（线画粗一点）

### 类型 B：关键均线

**定义**：日线图上的以下两条均线当前所在的价格：

- **MA50**：过去 50 根日线收盘价的算术平均值
- **MA200**：过去 200 根日线收盘价的算术平均值

这两个数值在任何交易所/TradingView 上都可以直接显示，无需手动计算。

**作为关键位的附加条件**：

- 均线必须**走平或朝有利方向倾斜**
  - 做多时：MA 走平或向上 → 有效。MA 明显向下 → 不作为支撑使用
  - 做空时：MA 走平或向下 → 有效。MA 明显向上 → 不作为阻力使用
- "明显向下/向上"的量化判断：**MA 当前值 vs 10 天前的值，差异 > 3%**

### 类型 C：成交密集区

**定义**：日线图上，过去 90 根K线内，存在一段时期满足：

- 连续 ≥ 10 根K线 的最高价和最低价都落在某个价格区间内
- 这个区间的振幅 ≤ **8%**（即区间高点比低点高不超过 8%）

该区间的**上沿**是阻力位，**下沿**是支撑位。

**举例**：

ETH 在2月1日~2月15日（15根日线）：
期间最高价 2,650，最低价 2,480
振幅 = (2650 - 2480) / 2480 = 6.9% < 8% ✓
持续 15 天 > 10 天 ✓
→ 2,650 是阻力位，2,480 是支撑位

## 三、确认K线的精确定义

以下所有定义基于**日线K线收盘后**判断。用 O=开盘价、H=最高价、L=最低价、C=收盘价。

### 做多确认K线（3种，满足任一即可）

#### ① 锤子线

条件（全部满足）：

1. C > O（收阳线）或 C ≈ O（十字星，|C-O| < 总振幅的5%）
2. 下影线长度 ≥ 实体长度的 2 倍
   下影线 = min(O,C) - L
   实体 = |C - O|
   即：(min(O,C) - L) ≥ 2 × |C - O|
3. 上影线 ≤ 实体长度
   上影线 = H - max(O,C)
   即：(H - max(O,C)) ≤ |C - O|

直白说：下面一根长"尾巴"，上面基本没影线，说明空头把价格打下去后被多头拉回来了。

#### ② 看涨吞没

条件（全部满足）：

1. 前一根K线是阴线：前C < 前O
2. 当前K线是阳线：C > O
3. 当前K线的实体完全覆盖前一根的实体：
   C > 前O（当前收盘 > 前一根开盘）
   O < 前C（当前开盘 < 前一根收盘）
4. 当前K线实体长度 > 前一根实体长度
   |C - O| > |前C - 前O|

直白说：一根大阳线把前一根阴线"吃掉"了。

#### ③ 早晨之星

条件（需要看最近3根K线，按时间顺序为 K1、K2、K3）：

1. K1 是阴线：K1.C < K1.O
2. K2 是小实体：|K2.C - K2.O| < K1 实体长度的 30%
3. K2 的实体位于 K1 实体下方：max(K2.O, K2.C) < K1.C
4. K3 是阳线：K3.C > K3.O
5. K3 收盘价至少进入 K1 实体的 50% 以上：
   K3.C > K1.O - (K1.O - K1.C) × 0.5
   即 K3.C > (K1.O + K1.C) / 2

直白说：大阴线 → 小十字星（犹豫）→ 大阳线拉回。

### 做空确认K线（3种，满足任一即可）

以上三种形态的**镜像**：

| 做多形态 | 对应做空形态 | 区别                     |
| -------- | ------------ | ------------------------ |
| 锤子线   | 射击之星     | 长上影线在上面，下影线短 |
| 看涨吞没 | 看跌吞没     | 阴线吃掉前一根阳线       |
| 早晨之星 | 黄昏之星     | 大阳线→小实体→大阴线拉回 |

#### 射击之星的精确条件

1. C < O（收阴线）或 |C-O| < 总振幅的5%（十字星）
2. 上影线 ≥ 实体的 2 倍：(H - max(O,C)) ≥ 2 × |C - O|
3. 下影线 ≤ 实体：(min(O,C) - L) ≤ |C - O|

#### 看跌吞没的精确条件

1. 前一根是阳线：前C > 前O
2. 当前是阴线：C < O
3. O > 前C 且 C < 前O（当前实体完全覆盖前一根）
4. |C - O| > |前C - 前O|

#### 黄昏之星的精确条件

K1阳线 → K2小实体(在K1上方) → K3阴线(收盘深入K1实体50%以下)
（早晨之星的镜像，方向全部反转）

## 四、入场条件汇总（检查表）

### 做多入场

□ BTC 方向过滤：BTC 强势或震荡（允许做多）
□ 当日最低价 ≤ 关键支撑位 × 1.02（触及支撑位）
□ 关键位类型：□水平支撑 □MA50/200（走平或上行） □成交密集区下沿
□ 日线收盘后出现确认K线：□锤子线 □看涨吞没 □早晨之星
□ 确认K线的收盘价 > 关键位价格（支撑未被有效跌破）
□ 当日24h成交量 ≥ 200万 USDT
□ 止损幅度在 3%~15% 之间
□ 仓位已计算，最大亏损 ≤ 总资金 2%
□ 风控检查清单 8 条全部通过
→ 全部打勾 → 入场

### 做空入场

□ BTC 方向过滤：BTC 弱势或震荡（允许做空）
□ 当日最高价 ≥ 关键阻力位 × 0.98（触及阻力位）
□ 关键位类型：□水平阻力 □MA50/200（走平或下行） □成交密集区上沿
□ 日线收盘后出现确认K线：□射击之星 □看跌吞没 □黄昏之星
□ 确认K线的收盘价 < 关键位价格（阻力未被有效突破）
□ 当日24h成交量 ≥ 200万 USDT
□ 止损幅度在 3%~15% 之间
□ 仓位已计算，最大亏损 ≤ 总资金 2%
□ 风控检查清单 8 条全部通过
→ 全部打勾 → 入场

## 五、止损位设定

| 方向 | 止损位                | 计算方式             |
| ---- | --------------------- | -------------------- |
| 做多 | 关键支撑位 × (1 - 2%) | 支撑位下方留 2% 缓冲 |
| 做空 | 关键阻力位 × (1 + 2%) | 阻力位上方留 2% 缓冲 |

**举例（做多）**：

关键支撑位 = 60,000
止损价 = 60,000 × 0.98 = 58,800
入场价 = 60,600（确认K线收盘价）
止损幅度 = (60,600 - 58,800) / 60,600 = 3.0%

---

## 六、下单方式

- 确认信号在日线收盘后才能确定
- 入场时机：**确认K线收盘后的次日**，在确认K线收盘价附近限价挂单，或开盘市价入场
- 入场后立即挂止损单

## 七、完整入场示例

周日标记：BTC 60,000 是关键支撑位（过去3个月2次触及）

周三晚上看盘：
BTC 当前价 60,500，距关键位 0.8% → 进入 ±2% 范围 ✓

日线收盘后检查K线：
O=60,800 H=61,200 L=59,200 C=60,600
实体 = |60,600 - 60,800| = 200
下影线 = min(60,800, 60,600) - 59,200 = 60,600 - 59,200 = 1,400
上影线 = 61,200 - max(60,800, 60,600) = 61,200 - 60,800 = 400

    锤子线检查：
      下影线(1,400) ≥ 实体(200) × 2 = 400 → 1,400 ≥ 400 ✓
      上影线(400) ≤ 实体(200) → 400 ≤ 200 ✗
    → 不是标准锤子线（上影线太长）

    但收的是阴线，检查是否构成其他形态...
    前一根K线：前O=61,500 前C=60,900（阴线）
    当前：O=60,800 C=60,600（阴线）
    → 也不是看涨吞没（当前不是阳线）

    → 今天没有确认信号 → 不入场，继续等

周四晚上看盘：
O=60,500 H=61,800 L=60,200 C=61,500
前一根：前O=60,800 前C=60,600（阴线）
当前是阳线 ✓
C(61,500) > 前O(60,800) ✓
O(60,500) < 前C(60,600) ✓
当前实体(1,000) > 前实体(200) ✓
→ 看涨吞没 ✓

    收盘价 61,500 > 关键位 60,000 ✓

    止损 = 60,000 × 0.98 = 58,800
    止损幅度 = (61,500 - 58,800) / 61,500 = 4.4%（在3%~15%范围内 ✓）

    仓位计算（500U总资金，5x杠杆）：
    保证金 = 10 / (4.4% × 5) = 45.5U
    最大亏损 = 10U = 2% ✓

    检查清单 8 条 → 全通过 ✓

周五：
限价 61,500 挂买入单
同时挂止损卖出 58,800
→ 入场完成
